# Инструкция по развертыванию

## Локальный запуск

### Вариант 1: Простое открытие в браузере
1. Скачайте все файлы в одну папку
2. Дважды кликните на `index.html`
3. Или откройте браузер и перетащите файл в окно

### Вариант 2: Локальный сервер (рекомендуется)
```bash
# Перейдите в папку с проектом
cd /путь/к/проекту

# Запустите сервер
python -m http.server 8000

# Откройте в браузере
http://localhost:8000
```

### Вариант 3: Node.js
```bash
# Установите http-server глобально
npm install -g http-server

# Запустите
http-server -p 8000
```

## Продакшен развертывание

### Статический хостинг
Платформа полностью клиентская и может быть размещена на любом статическом хостинге:

- GitHub Pages
- Netlify
- Vercel
- Firebase Hosting
- AWS S3 + CloudFront

### Пример для GitHub Pages
1. Создайте репозиторий на GitHub
2. Загрузите файлы
3. В настройках репозитория включите GitHub Pages
4. Выберите source: "Deploy from a branch"
5. Выберите branch: "main" и folder: "/ (root)"
6. Сайт будет доступен по адресу: `https://username.github.io/repository-name`

### Пример для Netlify
1. Перетащите папку с файлами на https://app.netlify.com/drop
2. Сайт сразу будет доступен
3. Или подключите Git-репозиторий для автоматического деплоя

### Пример для Vercel
```bash
# Установите Vercel CLI
npm i -g vercel

# Разверните
vercel --prod
```

## Docker (опционально)

Создайте `Dockerfile`:

```dockerfile
FROM nginx:alpine
COPY . /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

Соберите и запустите:
```bash
docker build -t stress-test-app .
docker run -p 8080:80 stress-test-app
```

## Требования

- Современный браузер с поддержкой ES6+
- Интернет-соединение для загрузки Chart.js (CDN)
- JavaScript включен в браузере

## Проверка после развертывания

1. Откройте сайт в браузере
2. Проверьте, что загружается 5 экранов
3. Введите тестовые данные:
   - Доход: 80000
   - Обязательные расходы: 40000
   - Переменные расходы: 20000
   - Подушка: 200000
4. Перейдите к сценариям
5. Запустите симуляцию
6. Проверьте, что отображаются график и таблицы
7. Перейдите к вердикту
8. Проверьте индекс стресс-устойчивости
9. Сохраните прогон
10. Проверьте историю

## Настройка

Все настройки в коде:
- `scenarios.js` - параметры сценариев
- `styles.css` - визуальное оформление
- `app.js` - логика интерфейса

## Поддержка

Если что-то не работает:
1. Откройте консоль браузера (F12 → Console)
2. Проверьте ошибки
3. Убедитесь, что Chart.js загружается (нет блокировки CDN)
4. Проверьте, что JavaScript включен
